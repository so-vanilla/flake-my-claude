# /org-remind - 生活リマインド

日常の消耗品・物品の廃棄/交換/寿命をLLMが推定し、`~/org/todo.org` にTODOを追記する。

## 引数

`$ARGUMENTS`: `food` / `bottle` / `clothes` / 省略可（省略時はカテゴリ選択）

## 処理フロー

以下のステップを順に実行すること。

### 1. フォーマットルールの確認

Read ツールで `~/.claude/commands/org-format-rules.md` を読み込み、フォーマットルールを把握する。

### 2. カテゴリの判定

- `$ARGUMENTS` が `food` / `bottle` / `clothes` のいずれかであればそのカテゴリに進む
- 省略または不明な場合は AskUserQuestion でカテゴリを選択する:
  - `food` — 食品の開封後廃棄タイミング
  - `bottle` — ボトル（洗剤・化粧品等）の交換時期
  - `clothes` — 衣類の寿命・買い替え目安

### 3. 情報収集（カテゴリ別）

#### food の場合

AskUserQuestion で以下を収集する（複数アイテムを1回で受け付ける）:
- 食品名（例: 醤油、豆腐、カット野菜）
- 保存方法（冷蔵 / 冷凍 / 常温）
- 開封日（YYYY-MM-DD 形式）

#### bottle の場合

AskUserQuestion で以下を収集する:
- ボトル名（例: ヘアトリートメント、食器用洗剤）
- 使用開始日（YYYY-MM-DD 形式）
- 交換目安ヶ月数（LLMが推定して提案し、ユーザーが確認・修正できるようにする）

#### clothes の場合

AskUserQuestion で以下を収集する:
- アイテム名（例: 白Tシャツ、ウールニット）
- カテゴリ（Tシャツ / ニット / パンツ / コート / インナー 等）
- 素材（綿 / ウール / ポリエステル / 混紡 等）
- 洗濯頻度（週1 / 2週に1 / 月1 / シーズン1 等）
- 購入日（YYYY-MM-DD 形式、省略可）

### 4. DEADLINEの計算

収集した情報をもとに以下を決定する。**曜日は必ず `date -d "YYYY-MM-DD" "+%a"` で取得すること（推測禁止）。ゼロパディング必須。**

#### food の廃棄目安

食品名と保存方法から廃棄目安日数を推定し、`開封日 + 目安日数` のDEADLINEを算出する。

参考目安（LLMが適宜調整すること）:

| 種別 | 保存方法 | 目安 |
|---|---|---|
| 豆腐・木綿/絹 | 冷蔵 | 3〜5日 |
| 納豆 | 冷蔵 | 3〜5日 |
| 卵（開封後） | 冷蔵 | 2週間 |
| カット野菜 | 冷蔵 | 2〜3日 |
| 醤油（開封後） | 冷蔵 | 1〜2ヶ月 |
| みそ（開封後） | 冷蔵 | 2〜3ヶ月 |
| 牛乳（開封後） | 冷蔵 | 2〜3日 |
| 肉・魚（生） | 冷蔵 | 1〜2日 |
| 肉類 | 冷凍 | 1〜2ヶ月 |
| 野菜類 | 冷凍 | 1ヶ月 |
| 乾麺・乾物 | 常温 | 開封後1〜2ヶ月 |

DEADLINEはワンショット（リピーターなし）。

#### bottle の交換目安

ボトル名から交換目安を推定し、`使用開始日 + 交換目安` のDEADLINEを算出する。

参考目安（LLMが適宜調整すること）:

| 種別 | 交換目安 |
|---|---|
| シャンプー・コンディショナー | 2〜3ヶ月 |
| ヘアトリートメント | 3〜4ヶ月 |
| ボディソープ | 1〜2ヶ月 |
| 食器用洗剤 | 2〜3ヶ月 |
| 洗濯洗剤（液体） | 3〜4ヶ月 |
| ハンドソープ | 1〜2ヶ月 |

リピーターは `+Nm`（完了した日から次回設定）を使用。`++Nm`（固定スケジュール）は使わない。

#### clothes の寿命目安

素材・カテゴリ・洗濯頻度から何シーズン使えるかを推定し、寿命終了の目安日を算出する。

参考目安（LLMが適宜調整すること）:

| カテゴリ | 素材 | 洗濯頻度 | 目安シーズン |
|---|---|---|---|
| Tシャツ | 綿 | 週1 | 4〜6シーズン（2〜3年） |
| Tシャツ | ポリエステル | 週1 | 6〜8シーズン（3〜4年） |
| ニット | ウール | 月1 | 8〜12シーズン（4〜6年） |
| パンツ（デニム） | 綿 | 2週に1 | 6〜10シーズン（3〜5年） |
| コート | ウール | シーズン1 | 10〜16シーズン（5〜8年） |
| インナー | 綿 | 週2〜3 | 2〜4シーズン（1〜2年） |

DEADLINEはワンショット（リピーターなし）。

### 5. ユーザー確認

追記予定のTODOを表示し、AskUserQuestion で確認:
- そのまま追記
- 修正して追記
- スキップ

### 6. todo.org への追記

Read ツールで `~/org/todo.org` の現在の内容を確認後、ファイル末尾に追記する。

#### food のフォーマット:

```org
* TODO [食品名]を廃棄確認    :@life:food:
DEADLINE: <YYYY-MM-DD Www>
```

#### bottle のフォーマット:

```org
* TODO [ボトル名]のボトル交換    :@life:household:
DEADLINE: <YYYY-MM-DD Www +Nm>
```

#### clothes のフォーマット:

```org
* TODO [アイテム名]の状態確認・買い替え検討    :@life:clothes:
DEADLINE: <YYYY-MM-DD Www>
```

### 7. タグ候補の案内

追記後、以下のタグ候補をユーザーに案内する（初回使用のタグを含む場合）:

```
org-mode タグ候補:
  :@life:     - 生活全般（コンテキストタグ）
  :food:      - 食品・消費期限管理
  :household: - 日用品・消耗品
  :clothes:   - 衣類・被服
  :seasonal:  - 季節管理
```

拡張カテゴリ候補（将来的に `/org-remind` に追加可能）:
- `medicine` — 薬・サプリの期限管理
- `appliance` — 家電メンテナンス（フィルター清掃・電池交換）
- `plant` — 植物の水やり・肥料タイミング
- `clean` — 定期掃除タスク（排水溝・エアコンフィルター等）

### 8. 結果報告

追記したTODOの一覧（タイトル・DEADLINE・タグ）を報告する。
