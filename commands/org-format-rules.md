# org-mode フォーマットルール

このファイルは org-mode ファイルを編集するスキルのためのガードレール文書です。
**直接コマンドとして呼び出すものではありません。**

org-mode ファイルを編集する際は、必ず以下のルールに従うこと。

---

## タイムスタンプ記法

### 基本フォーマット

| 種別 | 記法 |
|---|---|
| 日付のみ | `<YYYY-MM-DD Www>` |
| 日付+時刻 | `<YYYY-MM-DD Www HH:MM>` |
| 時間範囲 | `<YYYY-MM-DD Www HH:MM-HH:MM>` |
| リピーター付き | `<YYYY-MM-DD Www HH:MM ++1d>` |

### ゼロパディング（必須）

月・日・時・分はすべて **2桁ゼロパディング** すること。

```
正: <2026-04-05 Sun 09:00>
誤: <2026-4-5 Sun 9:0>

正: <2026-01-12 Mon 08:30-09:30>
誤: <2026-1-12 Mon 8:30-9:30>
```

### 曜日の計算（LLM推測禁止）

曜日は **必ずコマンドで取得** すること。LLMによる暗算・推測は禁止。

```bash
date -d "YYYY-MM-DD" "+%a"
# 例: date -d "2026-04-12" "+%a"  → Sun
```

曜日略称: `Mon`, `Tue`, `Wed`, `Thu`, `Fri`, `Sat`, `Sun`（英語3文字）

---

## DEADLINE / SCHEDULED 記法

```
DEADLINE: <YYYY-MM-DD Www>
DEADLINE: <YYYY-MM-DD Www HH:MM>
DEADLINE: <YYYY-MM-DD Www HH:MM ++1d>
SCHEDULED: <YYYY-MM-DD Www>
```

- `DEADLINE:` の後はスペース1つのみ
- 山括弧 `<>` で囲む（角括弧 `[]` は非アクティブタイムスタンプで別意味）

---

## リピーター記法

| 記法 | 意味 |
|---|---|
| `++1d` | 完了日に関わらず、次の繰り返し日を1日後に更新 |
| `++1w` | 同上、1週間後 |
| `++1m` | 同上、1ヶ月後 |
| `+1d` | 完了した日から1日後に設定 |
| `.+1d` | 完了した時刻から1日後に設定 |

`++` と `+` は意味が異なるため、既存の記法を変更しないこと。

---

## 見出し構造

```org
* TODO タイトル
* DONE タイトル
** サブ見出し
```

- `*` の後にスペース1つ、TODOキーワードの後にスペース1つ
- インデントにタブは使わない

---

## 変更禁止ブロック

以下のブロックは **構造を壊さないこと**。内部のプロパティ値を変更する場合も、ブロック自体の開始・終了行は保持する。

```org
:PROPERTIES:
:ID: ...
:CUSTOM_ID: ...
:END:
```

```org
:LOGBOOK:
- State "DONE" from "TODO" [2026-03-01 Sun 10:00]
:END:
```

---

## 空行ルール

- トップレベル見出し（`* ...`）の間には空行1行
- 削除操作後に空行が2行以上連続した場合は1行にまとめる
