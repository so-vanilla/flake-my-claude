# /org-bump - デイリータスクのDEADLINE修正

`~/org/todo.org` のデイリーリピートタスク（`++1d`）で DEADLINE が過去日のものを、今日または明日に修正する。

**確認なしで即実行し、結果を事後報告する。**

## 引数

なし

## 処理フロー

以下のステップを順に実行すること。

### 1. todo.org の読み込み

- Readツールで `~/org/todo.org` を読み込む

### 2. 対象タスクの抽出

以下の全条件を満たすエントリを抽出:

- ステータスが `TODO`
- DEADLINE行に `++1d` を含む
- DEADLINE の日付が今日より前（過去日）

### 3. 新しい日付の決定

各タスクについて、DEADLINE行の時刻部分と現在時刻を比較:

- **deadline時刻 > 現在時刻** → 今日の日付に設定
- **deadline時刻 <= 現在時刻** → 明日の日付に設定
- **時刻指定なし** → 今日の日付に設定

### 4. DEADLINE行の書き換え

- Editツールで各タスクの DEADLINE行の日付部分のみを書き換える
- `++1d` やその他の修飾子はそのまま維持する
- 例: `DEADLINE: <2025-01-01 Wed 09:00 ++1d>` → `DEADLINE: <2025-02-18 Tue 09:00 ++1d>`

### 5. 結果報告

以下の情報を報告:

- 対象タスクの総数
- 各タスクのタイトルと変更内容（旧日付 → 新日付）
- 対象タスクがなかった場合は「更新対象のデイリータスクはありません」と報告
